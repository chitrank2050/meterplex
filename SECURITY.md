# Security Policy 🛡️

## Supported Versions

We actively provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| >= 1.0.0| :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take the security of Meterplex seriously. If you believe you have found a security vulnerability, please **DO NOT** open a public issue. Instead, please report it through one of the following channels:

1. **GitHub Private Vulnerability Reporting**: Use the "Report a vulnerability" button in the Security tab of this repository.
2. **Email**: Send a detailed report to `security@meterplex.com` (replace with your actual email).

### What to include in your report

- A description of the vulnerability.
- Steps to reproduce the issue.
- Potential impact if exploited.
- Any suggested fixes or mitigations.

## Our Process

1. **Acknowledgment**: We will acknowledge your report within 48 hours.
2. **Evaluation**: We will evaluate the report and determine the severity.
3. **Fix**: If valid, we will work on a fix and release it as soon as possible.
4. **Disclosure**: We will coordinate with you on the timing of a public advisory.

Thank you for helping keep Meterplex secure! 🌊🛡️
